difference between restore & reset
====================================
The main difference between the two commands is that git restore is used for restoring files in the working tree from the index or another commit. This doesn't update the branch. Whereas, the git reset is used to update your brand.


need to learn
===============
git revert
git revert does this by creating a new commit that undoes the changes in the specified commit and so does not change the history.


fetch vs pull
===============
Git Fetch is the command that tells the local repository that there are changes available in the remote repository without bringing the changes into the local repository

pull ---- in this case we get changes from remote and directally merge that changes to local
fetch ---- in this case we get chnages from remote to local-git-log but these changes are not directally merge with local working directory

Stash
=======
git stash  ---------------create a backup of last changes and return back to last commit changes
git stash apply ----------------again apply last stash changes which we done last time
git stash list ----------showing all backups
git stash drop --------------delete last stash backups
git stash clear --------------delete all stash backups

git stash only backup that files which are already added or tracked file. it not create backup the newly created files.
git stash -u           -----------------also backup newly created file
git stash pop        ---------------apply and delete stash backup in one line.


git stash save "with any message"
git stash show stashName@{1}  ----------------show changes of any specific stash with it complete name with index
git stash apply stashName@{1}     ----------------apply any specific stash
git stash drop stashName@{1}     ----------------drop any specific stash
git stash branch branchName         ---------------apply the stash changes to any specific branch. like currently we are in master branch and we stash our code after some changes but we not want to apply that changes to current bransh . then with the help of "git stash branch branchName" command we apply changes to any other branch


Tag
========
Tag is just a alavel who point to a commit. means it refer to a commit or like a commit name.
Two types of tags:-
-simple tag
-annotated tag

-git tag mytag         ----------------------it create a tag with "mytag" name. means it create a tag of last commit. it creae the simple tag
-git tag --list          -----------------show the list of tags
-git tag --delete mytag      ----------------delete to a perticular tag

-git tag -a v.10 -------------- it create a annotated tag. when we enter this command with "-a" then we also have to add discription of the tag. which is not in simple tag.

-git diff v.10 v.11         ---------------get differnece b/w two tags
